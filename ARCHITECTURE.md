# AI 旅游场景规划 Agent — 架构方案

## 一、项目定位

构建一个智能旅游规划助手。用户输入目的地、天数、偏好等信息，Agent 自动生成完整的行程规划，涵盖景点、交通、餐饮、住宿、预算等维度的智能编排。

## 二、核心能力

| 能力 | 说明 |
|------|------|
| 多目的地行程编排 | 支持单城市/多城市连线，时间/距离感知的日程排布 |
| POI 智能推荐 | 基于用户偏好（美食/人文/自然/购物）推荐景点 |
| 酒店智能推荐 | 每天行程结束后基于当天活动区域和预算推荐酒店 |
| 预算优化 | 给定总预算，自动分配交通/住宿/门票/餐饮比例 |
| 实时感知 | 天气、节假日、旺季/淡季信息纳入规划 |
| 动态调整 | 支持用户反馈后 reroute 重新规划 |
| 交通衔接 | 城市间大交通 + 市内小交通的衔接计算 |

## 四、技术栈

| 层面 | 技术选型 |
|------|---------|
| AI/Agent 框架 | LangChain + LangGraph |
| 大语言模型 | Claude (通过 LangChain ChatAnthropic) |
| 后端框架 | Python FastAPI (async) |
| 数据存储 | MySQL 8.0+ |
| 缓存中间件 | Redis 7+ |
| 消息队列 | Kafka 3+ |
| 数据序列化 | Pydantic v2 |
| API 文档 | Swagger (FastAPI 自动生成) |
| 外部集成 | 高德地图 API / 和风天气 API |
| 部署 | Docker + Docker Compose |

## 五、Agent 工作流 (LangGraph)

```
1. GatherInfo   — 收集用户需求 (目的地/天数/偏好/预算)
     ↓
2. Research     — 并行查询天气、POI、交通信息
     ↓
3. Plan         — 生成初步行程框架 (天级别)
     ↓
4. Detail       — 细化每天的具体安排 (时段级别)
     ↓
5. Optimize     — 预算/时间合理性检查
     ↓
6. Present      — 格式化输出给用户
     ↓
7. Refine       — 根据用户反馈调整 (循环)
```

## 六、实现路径

### Phase 1 — 核心引擎
- 项目骨架搭建 (FastAPI + 数据模型 + 配置)
- MySQL 表结构设计与数据初始化 (5 城市 POI 数据)
- LangChain Agent + LangGraph 工作流实现
- 行程生成核心 API

### Phase 2 — 系统增强
- Redis 缓存接入 (会话/POI 缓存)
- Kafka 异步处理 (行程生成任务队列)
- 外部 API 集成 (地图/天气)
- 用户反馈与精调

### Phase 3 — 产品化
- Web UI
- 用户系统
- 多城市连线规划
- 性能优化与监控
